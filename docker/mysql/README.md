# docker/mysql/

MySQL 8 initialisation for the GnuKontrolR stack.

## init.sql

Runs once on first container start (via `docker-entrypoint-initdb.d`).
Creates the `webpanel` database and grants the `webpanel` user full access to it.

Customer databases are created dynamically by the panel API — each domain gets
its own MySQL schema inside the same MySQL instance, isolated by credentials.

## Credentials

Set via environment variables in `docker-compose.yml` (sourced from `.env`):

| Variable | Description |
|---|---|
| `MYSQL_ROOT_PASSWORD` | Root password (generated by installer) |
| `MYSQL_DATABASE` | Default database (`webpanel`) |
| `MYSQL_USER` | App user (`webpanel`) |
| `MYSQL_PASSWORD` | App user password (generated by installer) |

## Backups

```bash
bash setup.sh backup
```

Dumps all databases with `mysqldump --all-databases --single-transaction`.

## Connecting manually

```bash
docker compose exec mysql mysql -u root -p
```
